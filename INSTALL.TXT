1. As root edit your vim /etc/postgresql/*/main/pg_hba.conf file to include:

local   all         gridcalendar                      md5

2. Reload the configuration:

killall -HUP postgres

2a. Install vobject (needed for iCalendar):
# aptitude install python-vobject

3. Create the gridcalendar postgres user with password:

$ su
# su postgres
postgres$ createuser --pwprompt gridcalendar
Enter password for new role:
Enter it again:
Shall the new role be a superuser? (y/n) n
Shall the new role be allowed to create databases? (y/n) y
Shall the new role be allowed to create more new roles? (y/n) n
postgres$

4. Create the gridcalendar database:

postgres$ createdb -U gridcalendar -O gridcalendar gridcalendar
Password:
postgres$

6. Install these Debian packages to support geo functions:
libgdal1-1.5.0
postgresql-8.3-postgis
postgresql-server-dev-8.3
python-geoip   version higher then 1.2.1-2 in lenny, for example 1.2.4-1, because 1.2.1 doesn't have continent support
geoip-database

binutils
postgresql-8.3
python-psycopg2
python-setuptools

7. After instllation of PostGIS debian packages you must issue commands like this:
$ su
# su postgres
$ cd /usr/share/postgresql-8.3-postgis/
postgres@cloca:/usr/share/postgresql-8.3-postgis$ createlang plpgsql gridcalendar
postgres@cloca:/usr/share/postgresql-8.3-postgis$ psql -d gridcalendar -f lwpostgis.sql
postgres@cloca:/usr/share/postgresql-8.3-postgis$ psql -d gridcalendar -f spatial_ref_sys.sql
